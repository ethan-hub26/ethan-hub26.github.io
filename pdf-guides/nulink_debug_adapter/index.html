개발환경 구축
  1. Keil MDK-ARM을 설치하고 라이선스를 등록합니다.
  2. Pack Installer에서 Nuvoton 장치 팩을 설치합니다.
  3. Nu-Link 드라이버를 설치하고 BSP를 내려받아 샘플 프로젝트를 빌드합니다.

  • 최종 이미지는 ICP Tool에서 APROM에 기록하고 보존 옵션을 확인합니다.
1소개

Nu-Link Debug Adapter는 USB와 SWD를 기반으로 NuMicro 타깃을 디버깅하고 프로그램하는 범용 디버거/프로그래머입니다.

2하드웨어 사양

Nu-Link 계열은 USB 포트와 SWD 인터페이스를 통해 PC와 타깃 보드를 연결합니다.

모델에 따라 디버그, 온라인/오프라인 프로그래밍, 전압 설정, VCOM, Control Bus 지원 범위가 달라집니다.

Table 2-1 Nu-Link Adapter Function Comparison
Nu-Link-Pro

풀기능 모델. 오프라인 버튼과 전압 LED를 제공하며 SWD 전압을 소프트웨어로 바꿀 수 있습니다.

Nu-Link

기본형 모델. 디버깅과 온라인/오프라인 프로그래밍을 지원하며 SWD 기본 전압은 5.0V입니다.

Nu-Link-Me

보드 내장형 모델. VCOM을 제공하고 저항 설정으로 전압을 전환합니다.

Nu-Link2-Me

NuMaker 동봉형 모델. 온보드 SPI Flash 기반 오프라인 프로그래밍과 BUSY/START/PASS/FAIL 신호를 지원합니다.

Nu-Link-Pro는 USB 포트, 상태 LED, 오프라인 프로그래밍 버튼, SWD 포트, SWD I/O 전압 LED, SWD 전원 출력 LED를 포함하는 풀기능 모델입니다.

3주요 기능

Nu-Link는 디버그와 프로그래밍을 한 장치에 묶어 브레이크포인트, 레지스터 접근, Semihost, 온라인/오프라인 다운로드를 연속으로 처리합니다.

Table 3-1 Nu-Link Adapter Functions
디버깅
  • Keil MDK에서 브레이크포인트를 설정해 코드 흐름과 변수 상태를 점검합니다.
  • 직접 레지스터 제어 인터페이스로 레지스터 값을 읽고 써서 주변장치를 검증합니다.
  • Semihost를 사용하면 UART 배선 없이 호스트 로그를 확인할 수 있습니다.
프로그래밍
온라인 프로그래밍
  1. 호스트와 Nu-Link를 연결합니다.
  2. SWD로 타깃 보드를 연결합니다.
  3. 툴에서 Connect 후 타깃 칩 감지를 확인합니다.
  4. 감지 후 바로 다운로드를 시작합니다.
오프라인 프로그래밍
  1. PC에서 오프라인 이미지와 옵션을 Nu-Link에 저장합니다.
  2. 현장에서는 타깃만 연결하고 버튼으로 기록을 시작합니다.
  3. PASS/FAIL 상태를 LED 또는 제어 버스로 확인합니다.
소프트웨어 시리얼 번호

소프트웨어 시리얼 번호 기능으로 각 디바이스에 개별 식별값을 주입할 수 있습니다.

넓은 전압 범위와 드라이버
  • Nu-Link-Pro는 1.8V, 2.5V, 3.3V, 5.0V를 소프트웨어에서 선택합니다.
  • Nu-Link-Me와 Nu-Link2-Me는 점퍼/저항으로 전압 범위를 전환합니다.
  • Keil MDK와 IAR EWARM용 드라이버는 별도로 설치해야 합니다.
빨간 표식은 삽입된 브레이크포인트를 의미하고, 노란 화살표는 다음에 실행될 명령 위치와 PC 값을 보여줍니다.

4설치와 설정

Nu-Link는 USB와 SWD 사이의 브리지 역할을 하며, SWD 포트로 타깃 보드에 최대 5V / 500mA를 공급할 수 있습니다.

연결이 끝나면 ICP Tool, Keil MDK, IAR EWARM, NuEclipse GCC에서 같은 어댑터를 디버거/프로그래머로 사용할 수 있습니다.

ICP Tool
  1. ICP Tool을 설치하고 UI 언어와 타깃 칩을 선택합니다.
  2. Option 창에서 전압과 Offline Programming mode를 설정합니다.
  3. Connect를 눌러 Nu-Link와 연결합니다.
  4. 어댑터가 둘 이상이면 연결할 장치를 선택합니다.
  5. 칩이 감지되면 프로그래밍을 진행하고, 필요 없으면 Disconnect 또는 Stop Check로 해제합니다.
Keil MDK
  1. Keil MDK와 Keil용 Nu-Link 드라이버를 설치합니다.
  2. Project > Options for Target > Output에서 Debug Information을 켭니다.
  3. Debug 탭에서 Nuvoton Nu-Link M0 Debugger를 선택합니다.
  4. Settings에서 칩, Reset, IO Voltage를 점검합니다.
  5. Utilities에서 Use Target Driver for Flash Programming과 Update Target before Debugging을 활성화합니다.
IAR EWARM
  1. IAR EWARM과 IAR용 Nu-Link 드라이버를 설치합니다.
  2. Project > Options > General Options > Target에서 Nuvoton 디바이스를 선택합니다.
  3. Debugger > Setup에서 Third-Party Driver를 사용합니다.
  4. Download에서 flash loader와 .board 파일을 프로젝트 목적에 맞게 지정합니다.
  5. Third-Party Driver 페이지에서 Nu-Link_IAR.dll 경로를 지정하고 Port를 SWD로 설정합니다.
NuEclipse GCC
  1. Run > Debug Configuration에서 GDB Nuvoton NuLink Debugging 구성을 생성합니다.
  2. Main 탭에서 프로젝트명과 .elf 경로를 확인합니다.
  3. Debugger 탭에 nulink.cfg와 MCU 계열용 target cfg를 지정합니다.
  4. Startup 탭에서 Chip Series를 고르고 flash 또는 SRAM 로드 옵션을 선택합니다.
  5. Apply 후 Debug를 실행합니다.
Nu-Link는 USB와 SWD 사이의 브리지로 동작하며, 호스트 도구는 USB를 통해 타깃 칩을 디버그하고 프로그램합니다.

5부록

온라인 모드에서는 USB 전원을 사용하고, 오프라인 모드에서는 타깃 보드의 SWD VCC에서 전원을 받아 동작합니다. 전류는 모델과 전압 설정에 따라 달라집니다.

  • 온라인 예: Nu-Link-Pro 5.0V 101mA, Nu-Link 5.0V 110mA, Nu-Link-Me 3.3V 60mA
  • 오프라인 예: Nu-Link-Pro 5.0V 64mA, Nu-Link-Pro 1.8V 171mA, Nu-Link 5.0V 100mA
자동 IC 프로그래밍 시스템
  1. Nu-Link2-Me 전원을 켜면 START, BUSY, PASS, FAIL 신호가 초기화됩니다.
  2. START를 논리 0으로 내리면 프로그래밍이 시작됩니다.
  3. 작업 중 BUSY는 논리 0이며 토글될 수 있습니다.
  4. 종료 시 BUSY는 논리 1이 되고 PASS 또는 FAIL 중 하나가 논리 0이 됩니다.
자동 IC 프로그래밍 시스템에서는 SWD 핀과 함께 BUSY, START, PASS, FAIL 신호를 사용합니다.

BUSY가 1로 돌아오고 PASS가 0이 되는 경우 PASS 상태로 판정합니다.
